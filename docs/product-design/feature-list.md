# Vide Analyst - Feature List

> IDEO 디자인 문서 기반으로 추출한 전체 기능 목록.
> 각 기능은 해결하려는 사용자 Pain Point와 연결된다.

---

## Pain Point → Feature 매핑

| Pain Point | 핵심 기능 |
|------------|----------|
| "어디서부터 시작하지" (시작의 막막함) | 원클릭 검색, 분석 유형 선택, AI가 먼저 채움 |
| "숫자는 있는데 맥락을 모르겠다" | 종합 리포트, 강점/리스크 병렬, 경쟁사 포지셔닝 |
| "혼자 결정하는 게 두렵다" | 투자 대가 6인 라운드테이블, 관점 간 충돌 |
| "적정가를 모르겠다" | 인터랙티브 밸류에이션, 레인지 차트 |
| "기다리는 동안 지루하다" | 살아있는 로딩 (단계별 진행 + 스트리밍) |
| "분석 후 뭘 해야 하지" | 다음 행동 유도 (워치리스트, 알림, 공유) |

## Screen 1: 홈

### F-1. 티커 검색

| 항목 | 내용 |
|------|------|
| **설명** | 화면 중앙의 단일 검색창. 영문 티커(AAPL), 한글 회사명(애플), 영문 회사명(Apple) 모두 입력 가능 |
| **Pain Point** | 시작의 막막함 → 진입 마찰 제로 |
| **인터랙션** | 타이핑 즉시 자동완성 드롭다운. 종목 선택 시 분석 유형 선택 화면으로 이동 |
| **화면 참조** | Screen 1 |

### F-2. 트렌딩 종목

| 항목 | 내용 |
|------|------|
| **설명** | "지금 많이 찾는 기업" — 최근 분석 요청이 많은 종목 3~5개를 카드로 표시. 각 카드에 종목명, 등락률, 오늘 분석 건수 표시 |
| **Pain Point** | 뭘 분석할지 모르는 사람에게 시작점 제공 |
| **인터랙션** | 카드 클릭 시 해당 종목의 분석 유형 선택 화면으로 바로 이동 |
| **화면 참조** | Screen 1 |

### F-3. 최근 내 분석

| 항목 | 내용 |
|------|------|
| **설명** | 로그인 사용자의 최근 분석 히스토리. 종목명, 분석 일자, 핵심 결과 요약(대가 점수, 적정가) 한 줄로 표시 |
| **Pain Point** | 이전 분석을 다시 찾기 어려움 |
| **인터랙션** | "이어보기" 클릭 시 캐시된 분석 결과로 즉시 이동 |
| **화면 참조** | Screen 1 |
| **전제** | 로그인 상태. 비로그인 시 이 섹션 미표시 |

## Screen 2: 분석 유형 선택

| 항목 | 내용 |

| **Pain Point** | 내가 맞는 종목을 선택했는지 즉시 확인 |
| **데이터** | FMP Company Profile + Quote |
| **화면 참조** | Screen 2 |

### F-5. 분석 유형 선택 카드

| 항목 | 내용 |
|------|------|
| **설명** | 4개 분석 유형을 카드로 제시: (1) 종합 분석 리포트, (2) 투자 대가 관점, (3) 적정주가 분석, (4) 전체 분석 (All-in-One) |
| **Pain Point** | "내가 통제하고 있다"는 느낌. 필요한 분석만 선택 가능 |

| **화면 참조** | Screen 2 |

## Screen 3: 분석 진행

### F-6. 단계별 프로그레스 트래커

| 항목 | 내용 |
|------|------|
| **설명** | 분석 진행 상태를 단계별로 표시. 각 단계마다 체크마크(완료) 또는 스피너(진행 중) + 수집 결과 미리보기 텍스트 |
| **Pain Point** | "일이 진행되고 있다"는 신뢰감. 기다림을 기대로 전환 |
| **단계 목록** | 재무제표 수집 → 산업 데이터 수집 → 뉴스 수집 → 리포트 작성 → 투자자 분석(×6) → 밸류에이션 |
| **인터랙션** | 각 단계 완료 시 수집 결과 요약이 카드 안에 표시 ("매출 $394B, 영업이익률 30.7%") |
| **화면 참조** | Screen 3 |

### F-7. 리포트 실시간 스트리밍

| 항목 | 내용 |
|------|------|
| **설명** | LLM이 작성하는 리포트가 실시간으로 타이핑되듯 화면에 나타남. 하단에 프로그레스 바(%) 표시 |
| **Pain Point** | "AI가 지금 분석하고 있다"는 생생한 느낌 |
| **기술** | Edge Runtime + Vercel AI SDK streamText |
| **화면 참조** | Screen 3 |

### F-8. 로딩 트리비아

| 항목 | 내용 |
|------|------|
| **설명** | 분석 대기 중 화면 하단에 해당 종목과 관련된 투자 트리비아를 표시. ("버핏은 Apple을 '세계 최고의 비즈니스'라고 표현했으며...") |
| **Pain Point** | 대기 시간을 교육적 콘텐츠로 채움. 로딩이 "보너스 시간"이 됨 |
| **화면 참조** | Screen 3 |

## Screen 5: 투자 대가 분석

### F-14. 라운드테이블 합의 요약

| 항목 | 내용 |
|------|------|
| **설명** | 6명 투자자의 판정을 한 줄로 요약. 찬성/중립/반대 아이콘 + 합의 바 차트 ("4명 찬성 · 1명 중립 · 1명 반대") |
| **Pain Point** | 다양한 시각의 합의/불일치를 한 눈에 파악 |
| **화면 참조** | Screen 5 상단 |

### F-15. 투자자 개별 분석 카드

| 항목 | 내용 |
|------|------|
| **설명** | 각 투자자(버핏, 그레이엄, 린치, 달리오, 그린블랫, 피셔)별 독립 카드. 구성: (1) 아바타 + 이름 + 한 줄 소개, (2) 판정 (강력매수/매수고려/관망/매수부적합) + 점수(★), (3) 1인칭 화법 코멘트, (4) 기준별 점수 바 차트 (해자, 경영진, 재무 등), (5) 핵심 근거 불릿 + 리스크 불릿 |
| **Pain Point** | "혼자 결정하는 게 두렵다" → 6명의 전문가와 상의한 느낌 |
| **화면 참조** | Screen 5 |

### F-16. 관점 간 충돌 표시

| 항목 | 내용 |
|------|------|
| **설명** | 투자자 카드 하단에 "버핏과의 시각 차이" 같은 비교 섹션. 두 투자자가 왜 다른 결론에 도달했는지 한 문장으로 설명 |
| **Pain Point** | 의도적으로 불일치를 드러내 확증 편향 방지 |
| **화면 참조** | Screen 5 (그레이엄 카드 하단) |

### F-17. 투자 원칙 접기/펼치기

| 항목 | 내용 |
|------|------|
| **설명** | 각 투자자 카드 하단에 "이 분석의 근거가 된 [투자자]의 원칙 보기" 토글. 펼치면 해당 투자자의 핵심 투자 철학 설명 |
| **Pain Point** | 교육적 가치. "왜 이렇게 판단했는지" 학습 가능 |
| **화면 참조** | Screen 5 |

## Screen 6: 밸류에이션

### F-18. 종합 적정주가 레인지 차트

| 항목 | 내용 |
|------|------|
| **설명** | DCF, RIM, Comparable 결과를 하나의 가로 레인지 바에 통합 표시. 현재가 마커가 레인지 위에 표시. 안전마진 기준 매수가도 함께 표기 |
| **Pain Point** | 복수 밸류에이션의 결과를 "범위"로 사고하게 유도 |
| **인터랙션** | 레인지 바 색상이 현재가 위치에 따라 변화 (적정가 내 = Gold, 초과 = Terracotta) |
| **화면 참조** | Screen 6 상단 |

### F-19. DCF 인터랙티브 슬라이더

| 항목 | 내용 |
|------|------|
| **설명** | DCF 모델의 4개 핵심 가정치(매출성장률, 영업이익률, WACC, 영구성장률)를 각각 슬라이더로 조정 가능. AI가 초기 추천값을 세팅. 각 슬라이더 옆에 AI 코멘트 ("최근 3년 CAGR 8.1%, 서비스 성장 감안 8.5%") |
| **Pain Point** | "빈 엑셀의 공포" 완전 제거. AI가 먼저 채우고 사용자가 조정 |
| **인터랙션** | 슬라이더 이동 즉시 적정가 숫자가 실시간 재계산 (spring animation). 별도 "계산" 버튼 없음 |
| **기술** | LLM → 가정치 추론. TypeScript → 수치 계산. 슬라이더 조정 시 TypeScript만 재실행 (LLM 재호출 없음) |
| **화면 참조** | Screen 6 |

### F-20. RIM 분석

| 항목 | 내용 |
|------|------|
| **설명** | 잔여이익모델(Residual Income Model) 기반 적정주가. 자기자본, ROE, 자기자본비용을 기반으로 계산. DCF와 동일한 슬라이더 인터랙션 |
| **기술** | TypeScript로 순수 계산 |
| **화면 참조** | Screen 6 (스크롤) |

### F-21. Comparable 멀티플 비교

| 항목 | 내용 |
|------|------|
| **설명** | 동종업계 평균 P/E, EV/EBITDA 기준으로 산출한 적정주가. 경쟁사 리스트와 각사의 멀티플도 함께 표시 |
| **데이터** | FMP Peers + Ratios |
| **화면 참조** | Screen 6 (스크롤) |

### F-22. 시나리오 비교 테이블

| 항목 | 내용 |
|------|------|
| **설명** | 보수적 / AI 추천 / 낙관적 3개 시나리오의 적정주가를 나란히 표시. 각 시나리오의 핵심 가정치(성장률, 마진)도 함께 |
| **Pain Point** | 단일 숫자가 아닌 "범위"로 사고하게 유도 |
| **화면 참조** | Screen 6 하단 |
